Intro

The first hurdle you encounter when learning Linux is actually getting Linux installed on your computer. Installing a foreign operating system is daunting, and you are not willing to risk breaking your current Windows installation. Fortunately we can use a virtual machine to learn Linux without touching our current Windows installation.

In this tutorial we'll create a Linux virtual machine. Then you can experiment with Linux on the virtual machine, and if you break something you can spawn a new virtual machine with a fresh Linux install.

This tutorial is not meant to teach Linux. It is meant to give you access to a working Linux installation, and thus enable you to complete other Linux tutorials.

To work with Linux virtual machines in Windows, we'll install the following programs:

Notepad++
A text editor (similar to Windows Notepad) that recognizes both Linux-style line endings and Windows-style line endings. We'll use it to view and edit text files we create in the Linux virtual machine.
VirtualBox
Provides a platform for running virtual machines. This is the software that enables your computer to host virtual machines.
Vagrant
A command-line tool for quickly creating virtual machines (uses VirtualBox internally). We'll use this to get our virtual machine up and running quickly.
Cygwin
A Linux-like command line for Windows. We'll use this to run Vagrant on our Windows machine, and to access the command line on the Linux virtual machine.

Cygwin

Cygwin is different from the other program's we've installed so far. It is important for us to understand these differences.

Let's go over these differences before we install Cygwin.

Cygwin Intro

The first thing to understand is that Cygwin is not a single program. A Cygwin installation is a collection of many programs and files which work together to form the "Cygwin environment". We add programs and files to our Cygwin installation by installing "packages". Each package contains a collection of related programs and files which are installed (and uninstalled) together, as a single unit.

There are hundreds of packages available for Cygwin. Typically, you install packages on an as-needed basis, instead of installing all Cygwin packages at once. You can always install more packages later if you need them.

The Cygwin Setup utility manages the packages in your Cygwin installation. Run Cygwin Setup at any time to:

  • Add new packages to your Cygwin installation.
  • Update existing packages in your Cygwin installation.
  • Remove unneeded packages from your Cygwin installation.

Packages work together, and many packages depend on other packages in order to function. When a package "depends" on another package, it is said to have a "dependency" on the other package. A package's "dependencies" are all the other packages on which it "depends". Typically, you tell the Cygwin Setup utility which packages you want to install, and Cygwin Setup checks the packages you've selected to make sure all their dependencies are installed. If any dependencies need to be installed, Cygwin Setup offers to install the dependencies. This process is called "dependency resolution".

Road Map

To set up our Cygwin installation, we'll begin by downloading the Cygwin Setup utility (it's a single executable file). We'll place the Cygwin Setup utility in a permanent location, so we know where to find it in the future when we want to manage our Cygwin packages.

Next, we'll launch the Cygwin Setup utility.

Cygwin Setup will first download a list of available packages from the Internet. Then it will open a package browser, where we can search through the available packages and choose the packages we want to install (I'll give you a list of specific packages to install when we get to this step). Once we finalize our choices, Cygwin Setup will check if the packages we selected require any dependencies. Then, Cygwin Setup will download and install our selected packages and their dependencies.

Remember, you can always run Cygwin Setup again to add packages to your Cygwin installation, or remove packages from your Cygwin installation.

Let's get started.

Launch Cygwin Terminal

There are two ways to launch Cygwin Terminal:

  • Launch directly from the Windows start menu.
  • Launch through XWin Server (this is the xorg-server package we installed).

If we launch Cygwin Terminal directly from the Windows start menu, we will not be able to run GUI applications on our Linux virtual machine, we'll only be able to use the Linux command line. To run GUI applications, we must launch Cygwin Terminal through XWin Server.

Later in this tutorial, you'll be asked to launch Cygwin Terminal through XWin Server. Let's try this now, so we know how to do it later.

Launch XWin Server from the Windows start menu.

XWin Server starts up in the background. After it starts up, it adds two new icons to your Windows system tray:

  • XWin Server (X with an orange circle)
  • Cygwin/X (C with a green X)

Click on the Cygwin/X icon to bring up the Cygwin/X menu.

Choose System Tools > Cygwin Terminal from the menu to launch Cygwin Terminal through XWin Server.

A new Cygwin Terminal window appears.

Let's shut down XWin Server and Cygwin/X for now.

First, close the Cygwin Terminal window. After you close the Cygwin Terminal window, XWin Server and Cygwin/X are still running in the background.

To shut down XWin Server and Cygwin/X, click on the Cygwin/X icon in the system tray and select XDG Menu > Exit.

Cygwin Directory Structure

User Home Directories

Cygwin Setup creates a home directory for each Windows user on your computer. Home directories are a place for users to put their files and documents. Home directories appear under C:\cygwin\home. If your user name is "jdoe", then your Cygwin home directory is C:\cygwin\home\jdoe.

cygdrive Directory

Cygwin Setup creates a C:\cygwin\cygdrive directory. C:\cygwin\cygdrive contains shortcut links to each drive on your computer. For example, C:\cygwin\cygdrive\c is a shortcut to C:\. If you have a D:\ drive, you can access it via C:\cygwin\cygdrive\d. The shortcut links in C:\cygwin\cygdrive allow you to access all files on your computer from within the Cygwin environment.

Virtual Machine Setup

Road Map

To create our Linux virtual machine, we'll start by using Vagrant to generate a generic "Vagrantfile". A Vagrantfile is a configuration file that tells Vagrant what kind of virtual machine we want to create. We'll create a generic Vagrantfile using the vagrant init command on the command line. Then we'll use Notepad++ to edit a few lines in the Vagrantfile to customize our virtual machine.

Once we have our Vagrantfile, we'll run the vagrant up command to create the virtual machine. Vagrant reads the configuration in the Vagrantfile and creates the virtual machine in VirtualBox. Then Vagrant turns on the virtual machine. This is just like pressing the power button to turn on a real computer.

Once the virtual machine is up and running, we need to connect to it. We'll use "SSH" to connect to the virtual machine. SSH is used to access the command line of a remote computer over a network. We'll be accessing the command line of our virtual machine, which is actually running on our local computer, not somewhere else on the network. But, the virtual machine is just like a real computer, and your Windows operating system sees it as a separate, remote computer on the network. It's like there's a fake network between your Windows host, and the guest virtual machine. The network drivers that VirtualBox installed make this "virtual network" possible.

We'll run vagrant ssh to begin an SSH session on the virtual machine. During the SSH session, we are accessing the virtual machine's command line directly. All commands we enter on the command line are sent straight to the virtual machine, and run on the virtual machine. To test drive our virtual machine, we'll run a command to display the operating system version on the virtual machine.

After our quick test drive, we'll quit the SSH session and shut down the virtual machine. To quit the SSH session, we'll run exit to close the SSH connection and return to our local machine's command line. After we exit the SSH session, the virtual machine is still running in the background. We'll run vagrant halt to shut down the virtual machine. This is just like pressing the power button to shut down a real computer.

Let's get started.

Walkthrough

To start, launch Cygwin Terminal through XWin Server.

Before we run vagrant init to generate our generic Vagrantfile, we need to make sure Cygwin knows where to find the main vagrant.exe file. The vagrant.exe file is Vagrant's main executable file. It was installed when we installed Vagrant, and it contains the Vagrant program itself. To search for the vagrant.exe executable from Cygwin, run which vagrant.

If the terminal responds with /cygdrive/c/HashiCorp/Vagrant/bin/vagrant (or something similar) then Cygwin has successfully found vagrant.exe, and you can skip the next three paragraphs and jump to this paragraph. If, instead, we get no response, it means that Cygwin could not find vagrant.exe, and you must read on to solve the problem.

To tell Cygwin where to find vagrant.exe, we need to find the directory where vagrant.exe is installed, and add that directory to Cygwin's "PATH". The "PATH" is a special variable in the Cygwin command line environment. The "PATH" variable contains a colon-separated list of directories. When you type a command on the command line, Cygwin searches all the directories in the "PATH" for an executable file with the same name as the command you typed. You can see the directories in your "PATH" by running echo $PATH to output the colon-separated list of directories.

Let's add the directory that contains vagrant.exe to our path so we can run Vagrant from the Cygwin command line. First, use Windows File Explorer to find the vagrant.exe file. It should be installed in C:\HashiCorp\Vagrant\bin (which is the same as /cygdrive/c/HashiCorp/Vagrant/bin in Cygwin). Once you find the directory that contains vagrant.exe, add it to Cygwin's path by running PATH="/cygdrive/c/path/to/directory:$PATH". Of course, replace path/to/directory with the actual directory path. For instance, if vagrant.exe were located in /cygdrive/c/HashiCorp/Vagrant/bin, we would run PATH="/cygdrive/c/HashiCorp/Vagrant/bin:$PATH".

Now run echo $PATH to see the new directory prepended to your path. Then, run which vagrant to confirm that Cygwin can now find vagrant.exe.

When we start Cygwin, the command line places us in our Cygwin home directory (which is symbolized by ~). If we were to run vagrant init right now, it would create the generic Vagrantfile in our Cygwin home directory. Instead, let's put the Vagrantfile in a vm/linux subdirectory under our home directory.

First, run mkdir -p ~/vm/linux on the Cygwin command line to create the vm/linux subdirectory. Then, run cd ~/vm/linux to move to the vm/linux directory we just created. Now when we run vagrant init it will create the Vagrantfile in the vm/linux directory. Run vagrant init to generate a generic Vagrantfile in the vm/linux directory. Then open the vm/linux directory in Windows File Explorer to see the newly-created Vagrantfile.

Now we'll use Notepad++ to edit the generic Vagrantfile and customize our virtual machine configuration.

First, we'll tell Vagrant that we want to use the "ubuntu/trusty32" box as our virtual machine template. Let me explain this a bit.

Vagrant creates a virtual machine by cloning a "box". A "box" is a Vagrant virtual machine image. The "ubuntu/trusty32" box is a virtual machine image with Ubuntu installed. Ubuntu is a common Linux operating system. "Trusty" is a codename for Ubuntu version 14.04. There are many boxes to choose from, and you can even make your own. For now, we'll start with the "ubuntu/trusty32" box.

To use the "ubuntu/trusty32" box, we need to edit the Vagrantfile. Open the Vagrantfile in Notepad++, and change the line that reads config.vm.box = "base" to config.vm.box = "ubuntu/trusty32". This tells Vagrant to use the "ubuntu/trusty32" box as the template for our virtual machine.

Next, we need to tell Vagrant to allow X11 forwarding over SSH. X11 forwarding allows us to launch GUI applications during our SSH session. To do this, use Notepad++ to add a new line after config.vm.box = "ubuntu/trusty32". Type config.ssh.forward_x11 = true on the new line.

Finally, save the edited Vagrantfile.

Now it's time to create the virtual machine. Go to the Cygwin command line and run vagrant up.

Vagrant reads the Vagrantfile, and gets to work downloading and cloning the "ubuntu/trusty32" box. After the box is downloaded and cloned, Vagrant starts up the newly-created virtual machine.

Now that the virtual machine is up and running, we can connect to it via SSH. Run vagrant ssh to start a new SSH session on the virtual machine.

Now we are in an SSH session on the virtual machine, and we are accessing the virtual machine's command line directly. Any commands we type are sent directly to the virtual machine, and run on the virtual machine.

Let's check the operating system version on the virtual machine. To do this, run lsb_release -a.

The command runs on the virtual machine, and tells us that the operating system is Ubuntu 14.04.

Now, let's shut down the virtual machine. To shut down the virtual machine, we need to end our SSH session first.

Run exit to end the SSH session, and return to your local computer's command line.

Now that we're back on our local computer's command line, we can use Vagrant to shut down the virtual machine. Run vagrant halt to shut down the virtual machine.

Vagrant shuts down the virtual machine. Run vagrant status to verify that the virtual machine is shut down.

If we wanted to delete the virtual machine entirely, we could run vagrant destroy --force. Let's keep the virtual machine for now.

Virtual Machine Exploration

Road Map

In this section, we'll explore the files and folders on our virtual machine, and learn how to transfer files between our Windows host machine and the guest virtual machine.

We'll start this exploration by installing the following programs on the virtual machine:

PCManFM
A GUI application for browsing files and folders (similar to Windows File Explorer).
Leafpad
A GUI application for editing text files (similar to Windows Notepad).

After we install PCManFM and Leafpad, we'll launch PCManFM from the command line. Then we'll use PCManFM to take a quick look around the virtual machine's directory tree to convince ourselves that we are not looking at the files on our Windows computer.

Next, we'll use PCManFM to create a new text file on the virtual machine. Then we'll open the text file in Leafpad and type some text in the text file. At this point, we've created a new text file on our Linux virtual machine, using 100% Linux software.

After that, it's time to share the new text file with our Windows host machine. To do this, we'll copy the text file to a shared folder which is readable from both the Windows host and the guest virtual machine. Conveniently, Vagrant has already set up a shared folder for us: the vm/linux directory where we put our Vagrantfile is shared with the virtual machine. We can access this shared directory in the /vagrant directory on the virtual machine. We'll use PCManFM to copy our new text file to the /vagrant directory on the virtual machine. Then, on our Windows host, we'll open the vm/linux directory in Windows File Explorer to see the new text file.

Finally, we'll open the new text file in Notepad++ to see the text we typed in Leafpad on the virtual machine.

Let's get started.

Walkthrough

Complete the following steps using the skills you've learned:

  1. Start Cygwin Terminal through Cygwin/X.
  2. Change to the virt\linux directory (cd virt/linux).
  3. Start the virtual machine (vagrant up).
  4. SSH into the virtual machine (vagrant ssh).

Now we're ready to install Leafpad and PCManFM on the virtual machine. To do this, we'll use Ubuntu's package manager, which is called "APT". APT is similar to the Cygwin Setup program we used before: you select packages you want to install, the package manager resolves dependencies, and the package manager downloads and installs the packages and dependencies. Let's ask APT to install Leafpad and PCManFM.

First, we should make sure APT has downloaded the most up-to-date package metadata. (Recall that Cygwin setup performed a similar step.) To download the latest package metadata, run sudo apt-get update.

Hint: Installing packages requires administrator privileges. sudo is used to run a command with administrator privileges.

Run sudo apt-get install leafpad pcmanfm to ask APT to install Leafpad and PCManFM. APT resolves the dependencies for Leafpad and PCManFM, and confirms the installation.

Enter y to confirm that you would like to install Leafpad and PCManFM, as well as the listed dependencies. APT gets to work downloading and installing the required packages.

Wait for it to finish.

Leafpad and PCManFM are now installed on the virtual machine. Now we can run Leafpad and PCManFM from the command line. Let's run PCManFM to take a look at the files and folders on our virtual machine.

Run pcmanfm.

The PCManFM GUI appears in a new window. To display the folders as a tree, click on the Places dropdown.

Select Directory Tree.

Now we can see all the files and directories on the virtual machine. Browse around the files to convince yourself that these are not the files on your Windows computer.

Let's use Leafpad to make a new text file on our virtual machine. Navigate back to /home/vagrant in PCManFM, and right click to bring up a context menu.

Select Create New... > Empty File to create a new empty file.

Name the new file test-file.

Click to create the new file.

Let's edit test-file in Leafpad. Right click on test-file to bring up a context menu.

Select Leafpad from the context menu.

A new Leafpad window opens our empty test-file. Type something silly in Leafpad. I typed "Hello Windows! My name is Linux!".

Let's save the edited test-file. Click File > Save.

Let's share our new test-file with Windows. We can do this by copying test-file into a shared directory. Conveniently, Vagrant has already created a shared directory for us.

By default, Vagrant shares the directory that our Vagrantfile is in with the virtual machine. The shared directory is accessed at /vagrant on the virtual machine. Navigate to /vagrant in PCManFM. You can see our Vagrantfile there. This is the same Vagrantfile that is on our Windows host in the virt/linux directory, under our Cygwin home directory.

Use PCManFM to copy and paste our test-file from /home/vagrant to /vagrant. Open the virt/linux directory in Windows File Explorer to confirm that the new test-file is there. Now open the test-file in Notepad++ to see the message we wrote on the Linux virtual machine.

Conclusion

The skills you've learned in this article enable you to interact with Linux on Windows. No Linux tutorial is out of your reach now. You can run commands on your Linux virtual machine with reckless abandon. If something breaks, run vagrant destroy --force to destroy the virtual machine, then run vagrant up to create a fresh virtual machine.

Keep in mind, the files under your shared directory are accessible from the Linux virtual machine, so they will be wiped out if you run something like rm -r /vagrant to delete the /vagrant directory. If you do run destructive commands like that, only the files and directories under the shared directory will be wiped out. Be mindful of this, and watch where you place your Vagrantfiles. Any subdirectory below the Vagrantfile can be wiped out by running rm -r /vagrant on the virtual machine.
